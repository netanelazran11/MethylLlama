The Problem: Learning from Methylation Arrays

What is DNA methylation?

A chemical modification at CpG sites across the genome that acts as a stable, tissue-specific epigenetic switch — regulating gene expression without changing the DNA sequence itself.

Biological age, cancer, cell type identity, and disease state are all encoded in the methylation landscape.

Illumina Methylation Arrays

Measure the methylation level (0 = unmethylated → 1 = fully methylated) at hundreds of thousands of CpG sites across the genome. Hundreds of thousands of public samples exist, but most have no phenotype labels — making self-supervised pretraining essential.

The Opportunity

Self-supervised pretraining on unlabeled arrays — like BERT for text — can learn a general-purpose methylation encoder. The CLS token becomes a compressed representation of the entire methylation profile, useful for age prediction, cell-type deconvolution, and cancer classification.

The Challenge

In language models, word identity is already meaningful (semantics transfer). In methylation models, CpG identity embeddings start from random noise. The model must learn from scratch that nearby CpGs correlate, that promoter CpGs behave differently from gene-body CpGs, and so on.

Our idea: use a DNA model to give each CpG a biologically informed starting vector — before the model ever sees methylation data.

Why a DNA Model?

Methylation is a DNA phenomenon

DNA methylation occurs directly on the DNA strand at CpG dinucleotides. The identity of a CpG site is fully determined by its DNA sequence context: is it inside a CpG island? A promoter? A transcription factor binding site? A repeat element? All of this information lives in the DNA — not in RNA or expression data.

DNA is universal — computed once for all samples

Every human cell carries the same DNA sequence. CpG embeddings are extracted once per probe and reused across all 10,988 training samples. RNA expression is tissue- and condition-specific, so an RNA-based embedding would not serve as a stable, sample-independent initialization.

Why not RNA models?

RNA models (e.g., scBERT, Geneformer) encode gene expression — the downstream effect of methylation, not its cause. A promoter CpG controls whether a gene is transcribed; a DNA model captures that regulatory role directly.

What BMFM-DNA learned

BMFM-DNA (IBM, 113M parameters) was pretrained on the entire human genome (GRCh38) using a masked language modeling objective on DNA sequences. Through this training it learned rich biological patterns embedded in DNA:

  • CpG island structure — dense CpG regions that tend to stay unmethylated
  • Promoter and enhancer signatures — sequence motifs near transcription start sites
  • Transcription factor binding motifs — CTCF, SP1, and other regulators
  • Repeat element context — LINE-1, Alu, SINE regions that are often heavily methylated
  • Chromatin accessibility signals — open vs. closed chromatin is partly sequence-encoded

All of these patterns influence methylation — and BMFM-DNA encodes them into a single 768-dim vector per genomic window, before seeing any methylation measurements.

Step 1a: Files Needed to Create CpG Embeddings

Four resources are combined to map each CpG probe to a 768-dim DNA context vector

hg38 Reference Genome (GRCh38)

The full human genome sequence — ~3.2 billion base pairs across all chromosomes. Used to extract the exact DNA sequence surrounding each CpG site.

Why hg38? The most current and complete human assembly. BMFM-DNA was trained on hg38, so the same reference must be used to avoid coordinate mismatches.

HM450 Probe Manifest

A lookup table that maps each Illumina probe ID (e.g., cg00000292) to its exact chromosomal position in hg38 — chromosome name and base-pair coordinate.

Source: Zhou et al. 2017 (NAR) — the standard hg38-native annotation for the HM450 array. No coordinate conversion is needed.

BMFM-DNA Pretrained Model

IBM's SCModernBert model (113M parameters) pretrained on the full human genome. Reads a DNA sequence and outputs a 768-dim contextual embedding.

Used frozen — no fine-tuning, no weight updates. This is pure feature extraction: we only need its forward pass once per CpG.

AltumAge CpG List

The list of 21,368 CpG probe IDs present in our methylation dataset. This defines exactly which probes we need embeddings for — a subset of the full 450K array, filtered to the probes that appear in the AltumAge 21k training data.

How these four files connect

The CpG list defines which probes we need → the HM450 manifest tells us where each probe sits in the genome → hg38 gives us the DNA sequence at that location → BMFM-DNA converts that sequence into a 768-dim biological context vector.

Step 1b: Full Extraction Flow

For each of the 21,368 CpG sites — repeated in batches until all embeddings are produced

1

Look up the probe's genomic position

For each CpG probe ID, query the HM450 manifest to retrieve its chromosome and exact base-pair position in hg38 (e.g., chr6 : 29,910,668).

2

Extract the surrounding DNA window

Open the hg38 genome FASTA and read the 4,096 nucleotides centered on the CpG site (±2,048 bp). This wide window captures nearby promoters, CpG islands, regulatory motifs, repeat elements, and long-range chromatin context.

3

Encode with BMFM-DNA (frozen)

Feed the DNA sequence into the pretrained BMFM-DNA model. The model reads the nucleotide sequence and produces contextual representations for each position. We extract the hidden state at the CpG site token — the token whose span covers the CpG position — giving a 768-dim vector encoding the local genomic context at that exact site.

4

Save the embedding matrix

After processing all 21,368 probes, the result is a matrix of shape 21,368 × 768 — one row per CpG site, one column per embedding dimension. This file is saved once and reused across all training runs.

What each 768-dim vector encodes

  • CpG island, shore, and shelf membership
  • Promoter and enhancer proximity
  • Transcription factor binding motifs (CTCF, SP1, etc.)
  • Repeat element context (LINE-1, Alu, etc.)
  • Local GC content and sequence complexity

Key properties

  • Run once — no recomputation needed per training run
  • BMFM-DNA is fully frozen during extraction
  • All 21,368 probes have hg38 coordinates — 100% coverage
  • Special tokens (CLS, MASK, PAD) are never overwritten

Step 2: MethylLlama — Model Architecture

Information Flow

INPUT SEQUENCE CLS token + randomly selected 50% of CpG sites ~10,684 CpGs per forward pass
↓
EMBEDDING LAYER CpG identity (768-dim) + methylation level [0,1] ← BMFM-DNA vectors initialize the CpG identity part
↓
TRANSFORMER LAYERS (8×) Self-attention across all CpGs  |  RoPE position encoding
↓
CLS TOKEN — Global Methylation Profile 768-dim summary of the entire sample's methylation state
↓
DECODER — Predicts ALL 21,368 beta values

Key Design Choices

RoPE
Relative positional encoding within each attention layer — no fixed position table. Works correctly regardless of how many CpGs are in the sequence.
RMSNorm
Lightweight normalization applied before each sublayer (Pre-LN). More stable than standard BERT during early training.
SwiGLU
Gated feed-forward activation (same as LLaMA, GPT-4). More expressive than standard GELU — better at learning complex co-methylation patterns.
ScaleAdapt
Trainable continuous encoder for methylation levels [0,1]. Encodes the beta value as a rich signal rather than a single number — important because 0.85 and 0.87 are biologically distinct.

Architecture: MethylLlama

Every component was chosen to improve stability, efficiency, and the ability to capture fine-grained methylation signals

Component MethylLlama choice Why it matters
Normalization RMSNorm — ~40% faster, more numerically stable Cleaner gradients, less compute per layer
Layer order Pre-LN — normalize before each sublayer Stable training from step one, no explosion risk
Feed-forward SwiGLU — gated activation (same as LLaMA, GPT-4) More expressive at the same parameter count
Position encoding RoPE — relative, no fixed position table Handles any number of CpGs, generalizes to unseen lengths
Beta value encoder ScaleAdapt — trainable sinusoidal basis over [0,1] Fine-grained resolution — 0.85 and 0.87 are biologically distinct
Embedding scale Learned scale, initialized small (0.1) CpG identity learned gradually — prevents shortcut learning early on

How each CpG is represented

Each input token combines two things: a CpG identity vector (768-dim, initialized with BMFM-DNA — encodes the genomic context of that specific CpG site) and a methylation level signal (the measured beta value [0,1], encoded by ScaleAdapt). A small learned scale parameter controls the balance between these two components.

What changes vs. random initialization

🧭

Biological similarity from day one

Two CpGs in the same CpG island start with similar vectors, before the model has seen any methylation data. Random init gives them completely unrelated vectors.

⚡

Faster convergence expected

The model starts from a biologically informed prior and can focus on learning methylation dynamics — not on re-discovering basic genomic relationships from scratch.

🌍

Genome-scale context encoded

Each embedding carries the context of ±2,048 bp around the CpG: CpG island status, regulatory elements, transcription factor binding sites — all transferred from a 113M parameter DNA model.

🔓

Embeddings remain trainable

BMFM-DNA provides the starting point only. During pretraining the embeddings are updated and refined using the methylation data itself.

Why randomize the input every batch?

If the same CpGs were always the input, the model would memorize fixed mappings and stop generalizing. Random shuffling forces the model to learn global methylation patterns — it can never rely on any specific CpG always being available.

Why add contrastive loss?

Without it, the CLS token collapses — all samples produce nearly identical representations. Two random 50% views of the same sample → similar CLS. Views from different samples → pushed apart.
Loss = Reconstruction MSE + 0.1 × InfoNCE

Training setup

  • 300 epochs max, early stopping (patience 60)
  • Effective batch size: 256  |  2 × H200 GPUs
  • 7,416 train / 1,308 valid / 2,264 test samples

Why DNA init alone is not enough — what WCED must learn:

Tissue- and age-specific methylation

DNA sequence is the same in every cell. BMFM-DNA cannot learn that a CpG is methylated in blood but not in brain — this requires real methylation data across many samples.

Long-range co-methylation

CpGs far apart may be tightly co-regulated in aging tissue. These correlations are invisible to any local DNA window — only methylation data across thousands of samples reveals them.

How to use beta values

BMFM-DNA reads only DNA letters — no concept of methylation levels [0,1]. WCED teaches the model to combine CpG identity with methylation level into a meaningful global representation.

Fine-tuning: From Pretrained Encoder to Downstream Tasks

The pretrained CLS representation is the starting point for all downstream tasks

Biological Age Prediction

All 21,368 CpGs are provided as input (no masking at inference time). The CLS representation is passed to a lightweight prediction head to estimate biological age in years.

This directly tests whether the learned representation captures the aging methylation signal.

Cell Type Deconvolution

Predicts the proportion of different cell types in a mixed sample from its methylation profile. This tests whether CLS captures tissue identity — the key discriminative property we expect DNA init to improve.

What changes at fine-tuning time

  • All CpGs are used as input — no random masking
  • The reconstruction decoder is discarded
  • A task-specific prediction head is added to CLS
  • The full encoder is fine-tuned end-to-end
  • A lower learning rate is used to preserve pretrained representations

Why pretraining matters here

Fine-tuning datasets are typically small — often only hundreds of labeled samples. A pretrained encoder that already understands methylation patterns needs far fewer labeled examples to reach good performance, exactly like BERT fine-tuned on NLP tasks.

The hypothesis: DNA-initialized pretraining produces richer representations that transfer better to downstream tasks, especially in low-data settings.

What We Expect to See

⚡

Faster Convergence

DNA-init should reach low validation loss in fewer epochs — the model starts with biologically meaningful CpG similarities rather than learning them from scratch.

📈

Better Reconstruction Quality

Higher correlation between predicted and true beta values on held-out CpGs. We expect DNA-init to converge to a higher accuracy with fewer epochs.

🎯

More Discriminative Representations

Greater spread in the CLS embedding space — the encoder should separate cell types, tissues, and age groups more clearly than random init.

🏆

Better Fine-tuning Performance

The ultimate test: fine-tune both checkpoints on biological age prediction. If DNA-init wins, the genomic context prior is genuinely useful for methylation modeling.
